# AGENTS.md

Agent guide for **duckvep** — a DuckDB-native variant effect predictor (a
loadable DuckDB extension in Rust). Read [DESIGN.md](DESIGN.md) first; it is the
source of truth for architecture and the migration plan. This file is the
quick operational guide.

## What this is

A DuckDB extension (`duckdb-rs`) that reads genomics formats via **noodles**,
exposes the VEP consequence / HGVS / ACMG engine as SQL UDFs, and treats
annotation databases as plain Parquet/DuckDB tables joined by the optimizer —
instead of hand-rolled file formats. Successor to fastVEP (`../DuckfastVEP`).

## Build, test, run

```sh
make debug      # build/debug/duckvep.duckdb_extension (native, with metadata)
make test       # DuckDB sqllogictest suite (test/sql/*.test)
make readme     # render README.md from README.Rmd (live SQL via duckknit)
cargo fmt && cargo clippy   # keep clean before committing
```

- **DuckDB version is pinned.** The `duckdb-rs` crate targets **v1.5.3** with the
  *unstable* C ABI, so the built extension loads only in DuckDB v1.5.3. A
  matching CLI is at `.tools/duckdb` (gitignored). The system `duckdb` may differ
  and will refuse to load the extension.
- Loading an unsigned local build needs `-unsigned`:
  `./.tools/duckdb -unsigned -c "LOAD './build/debug/duckvep.duckdb_extension'; ..."`

## Layout

- `src/lib.rs` — extension entrypoint; registers functions.
- `src/io/` — genomics IO table functions (noodles). `vcf.rs`: `read_vcf`,
  `vcf_samples`. More readers (gff/fasta/bcf/...) go here.
- `src/vep/` — VEP compute UDFs (consequence/HGVS/ACMG), wrapping the fastVEP
  crates (planned; see DESIGN.md §6 wrap-then-delete).
- `test/sql/` — sqllogictest; `test/data/` — fixtures.
- `DESIGN.md` — design + design-review. `README.Rmd` → `README.md` (generated).

## Conventions

- **One design doc.** Put decisions in `DESIGN.md` + commit messages. No ADR
  file sprawl.
- **Edit `README.Rmd`, not `README.md`** — the `.md` is generated by `make readme`
  and its SQL outputs are real.
- **noodles sync readers only** (no tokio in the extension; DuckDB calls are sync).
- **Organism-agnostic:** join on natural `(chrom,pos,ref,alt)` columns; never a
  human-hardcoded packed key (see DESIGN.md §1).
- **Wrap-then-delete:** reach output parity with fastVEP before deleting any of
  the old hand-rolled formats.
- Match surrounding style; run `cargo fmt`/`clippy` clean.
